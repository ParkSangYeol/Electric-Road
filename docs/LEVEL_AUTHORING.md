# Electric Road 레벨 제작 가이드

## 목적

이 문서는 새로운 퍼즐을 만들거나 기존 퍼즐의 맵과 밸런스를 수정할 때 필요한 데이터와 검증 절차를 설명한다.

정식 레벨은 다음 위치에 있다.

```text
Assets/ScriptableObjects/PuzzleStage/
├─ Village/    # 1-1 ~ 1-10
├─ Town/       # 2-1 ~ 2-10
└─ City/       # 3-1 ~ 3-10
```

개발용 제작 씬은 `Assets/Scenes/Dev/StageBuilder.unity`다.

## 관련 구성 요소

| 구성 요소 | 역할 |
|---|---|
| `StageBuilder` | 빈 맵 생성, 기본형 최소 경로 탐색, 에셋 저장·로드 |
| `StageEditor` | ScriptableObject 맵을 UI 타일 그리드로 생성 |
| `StageArea` | 현재 그리드의 너비와 높이 보관 |
| `StageTile` | 씬에 생성된 한 칸의 타일·방향·전기 타입 |
| `StageScriptableObject` | 최종 스테이지 에셋 |

## 제작 전 결정할 항목

새 퍼즐을 만들기 전에 다음을 정한다.

1. 지역과 난이도 순서
2. 맵 크기
3. 하나 이상의 발전기 수와 위치
4. 공장 수와 위치
5. 장애물과 고정 경로
6. 사용할 스테이지 타입
7. 플레이어에게 해금할 팔레트 범위
8. 목표 해법과 예상 비용
9. 3성, 2성, 1성 비용 기준
10. 업적 필요 여부

## StageScriptableObject 필드

### 기본 정보

- `stageName`: UI 표시 이름이다. 정식 레벨은 `Village-1`, `Town-1`, `City-1` 형식을 사용한다.
- `stageType`: 런타임 전력 판정 방식을 선택한다.
- `width`, `height`: 맵 배열에서 자동 계산된다.

### 맵 정보

- `map`: 플레이어가 처음 보는 초기 상태
- `answerMap`: 제작자가 저장한 정답 상태
- `generatorPositions`: 초기 맵의 모든 발전기 좌표
- `generatorPos`: 기존 에셋 호환용 첫 발전기 좌표
- `numOfFactories`: 맵의 공장 수

`MakeMapByStageTiles`는 맵 크기, 모든 발전기 위치, 공장 수와 정답 비용을 계산한다.
런타임은 `generatorPositions`를 우선 사용하며 목록이 없는 기존 에셋에서만 `generatorPos`로 대체한다.
단, 데이터가 정확한지는 저장 후 인스펙터와 실제 플레이에서 다시 확인한다.

### 난이도와 비용

- `ans`: `answerMap`의 유료 타일 비용 합계
- `numOfPalate`: 팔레트 앞에서부터 활성화할 타일 종류 수
- `thresholds[0]`: 3성 최대 비용
- `thresholds[1]`: 2성 최대 비용
- `thresholds[2]`: 1성 최대 비용

임계값은 반드시 다음 관계를 만족해야 한다.

```text
thresholds[0] <= thresholds[1] <= thresholds[2]
```

`ans`가 자동으로 최적해를 보장하지는 않는다. 실제 가능한 더 저렴한 해법이 있는지 별도로 확인한다.

### 업적

- `isAchievement`: 퍼즐별 업적 사용 여부
- `achievementKey`: 플랫폼 업적 API 키
- `isCheapestClearType`: 특정 비용 이하일 때만 해금할지 여부
- `cheapestCost`: 비용 조건

Steam과 Stove에서 동일 키가 유효한지 플랫폼 설정과 대시보드를 함께 확인한다.

## 권장 제작 절차

### 1. 제작 씬 열기

`Assets/Scenes/Dev/StageBuilder.unity`를 연다.
기존 변경이 있는 씬이라면 먼저 현재 작업 내용을 확인하고 별도 복사본 사용을 고려한다.

### 2. 빈 맵 생성

너비와 높이를 입력하고 `StageEditor.CreateBlankStageInScene`을 실행한다.

현재 런타임 UI는 높이에 따라 다음 타일 프리팹을 선택한다.

- 높이 6: 작은 맵용 프리팹
- 높이 7: 중간 맵용 프리팹
- 높이 8: 큰 맵용 프리팹
- 그 외: 튜토리얼용 기본 프리팹

새 크기를 도입하면 화면 배치와 가이드 스케일을 함께 확인한다.

### 3. 초기 조건 배치

발전기, 공장, 장애물과 필요한 고정 타일을 배치한다.

- 발전기가 하나 이상인지 확인한다.
- 각 발전기는 독립된 전력망의 시작점으로 배치한다.
- 한 발전소의 경로가 다른 발전소를 통과하지 않는지 확인한다.
- 서로 다른 발전소의 경로가 전선, 특수 타일 또는 공장을 공유하지 않는지 확인한다.
- 공장 수가 의도와 일치하는지 확인한다.
- 플레이어가 수정해야 하는 칸은 초기 맵에서 빈 타일이어야 한다.
- 고정 타일의 방향과 전기 타입을 확인한다.

### 4. 정답 경로 작성

`StageBuilder.FindAnswer`를 실행하면 자동 계산 여부와 관계없이 현재 맵을 초기 맵으로 캡처한다.

공장이 하나인 기본형 퍼즐에서는 각 발전소를 시작점으로 우선순위 큐 탐색을 실행하고, 공장에 도달하는 가장 저렴한 후보를 적용한다. 경로는 다른 발전소를 통과하지 않는다.

공장이 여러 개면 자동 최단 경로 계산을 생략하고 비용 UI에 `Cost: Manual`을 표시한다. 이때 초기 맵은 이미 캡처되어 있으므로 씬에서 정답 경로를 수동으로 편집한 뒤 에셋을 저장한다.

다음 경우에는 자동 결과를 정답으로 신뢰하지 않는다.

- 증폭기 규칙
- 변환기 규칙
- 증폭기와 변환기 복합 규칙
- 공장이 여러 개인 맵
- 특수한 고정 경로 또는 전기 타입 상태가 있는 맵

`ApplyAmplifier`와 `ApplyModulator`는 현재 구현되지 않았으므로 해당 레벨은 수동 제작과 플레이 검증이 필요하다.

### 5. 에셋 저장

`StageBuilder.MakeStageFile`로 `StageScriptableObject` 에셋을 저장한다.
저장 전 `FindAnswer`를 실행해 초기 맵을 캡처해야 한다. 초기 맵이 없으면 저장이 중단된다.
공장이 여러 개인 맵은 초기 맵 캡처 후 수동 정답을 완성한 상태에서 저장한다.
정식 콘텐츠 디렉터리와 이름 규칙을 지킨다.

에셋 저장 후 다음 필드를 인스펙터에서 설정하거나 확인한다.

- `stageName`
- `stageType`
- `numOfPalate`
- `thresholds`
- 업적 관련 필드
- 모든 발전기 위치와 공장 수
- `ans`

### 6. 지역 데이터에 등록

해당 `GameStageScriptableObject.stageData` 목록의 올바른 순서에 새 에셋을 넣는다.

- Village: `VillageGameStage.asset`
- Town: `TownGameStage.asset`
- City: `CityGameStage.asset`

현재 UI와 다음 퍼즐 이동 로직은 지역당 10개를 전제로 한다. 개수를 변경하려면 UI 프리팹, `GameStageHandler`, `GameManager.LoadNextPuzzle`을 함께 수정한다.

### 7. 실제 플레이 검증

`PuzzleScene` 에셋을 직접 실행하는 것보다 메인 메뉴에서 지역과 퍼즐을 선택해 진입하는 흐름을 권장한다. `GameManager`가 현재 데이터를 설정해야 하기 때문이다.

## 밸런스 기준

정식 레벨의 기존 경향은 다음과 같다.

| 지역 | 스테이지 타입 | 맵 크기 | 공장 수 |
|---|---|---|---:|
| Village | DEFAULT | 6×6 ~ 8×8 | 1 ~ 2 |
| Town | DEFAULT | 6×6 ~ 8×8 | 3 |
| City | AMPLIFIER | 6×6 ~ 8×8 | 1 ~ 3 |

기존 레벨 대부분은 제작 정답 비용과 3성 기준이 같거나, 3성 기준에 소폭 여유를 둔다.
새 레벨도 정답 비용만 기계적으로 복사하지 말고 다음을 고려한다.

- 경로를 잘못 시작했을 때 생기는 교체 비용
- 분배기 대신 더 저렴한 공유 경로가 가능한지
- 코너 수를 줄이는 우회 경로가 가능한지
- 증폭기 위치를 바꿔 생기는 대체 경로
- 고정 타일을 의도하지 않은 방향에서 재사용할 수 있는지

## 필수 검증 체크리스트

### 데이터

- [ ] 에셋 이름과 `stageName`이 의도한 순서와 일치한다.
- [ ] `width`와 `height`가 실제 맵과 일치한다.
- [ ] 발전기가 하나 이상이며 모든 발전기 위치가 정확하다.
- [ ] `generatorPositions`가 초기 맵의 모든 발전기를 포함한다.
- [ ] 공장 수가 실제 맵과 일치한다.
- [ ] 모든 셀의 타일 참조가 비어 있지 않다.
- [ ] 고정 타일의 방향과 전기 타입이 정확하다.
- [ ] 별 임계값이 오름차순이다.
- [ ] `numOfPalate`가 필요한 타일을 해금한다.

### 플레이

- [ ] 의도한 정답으로 모든 공장에 전력이 도달한다.
- [ ] 각 발전소의 전력 탐색이 독립된 경로를 사용한다.
- [ ] 한 발전소의 경로가 다른 발전소를 경유하면 실패한다.
- [ ] 서로 다른 발전소의 경로가 같은 타일이나 공장을 방문하면 실패한다.
- [ ] 같은 발전소에서 갈라진 분기들이 다시 같은 타일로 합류하면 실패한다.
- [ ] 공장이 여러 개인 맵은 자동 경로 대신 수동 정답을 검증한다.
- [ ] 끊긴 경로는 실패한다.
- [ ] 잘못된 방향으로 연결하면 실패한다.
- [ ] 정확히 각 임계값인 비용에서 예상 별을 받는다.
- [ ] 세 번째 임계값을 초과하면 실패한다.
- [ ] Undo와 Redo 후 비용이 원래 값으로 돌아온다.
- [ ] 타일 교체와 삭제 후 비용이 정확하다.
- [ ] Draw, Select, Erase가 고정 타일을 잘못 수정하지 않는다.

### 진행도

- [ ] 최초 진입 시 올바른 퍼즐만 해금된다.
- [ ] 클리어 후 다음 퍼즐이 해금된다.
- [ ] 더 낮은 별로 재클리어해도 최고 기록이 유지된다.
- [ ] 마지막 퍼즐 이후 지역 화면으로 돌아간다.
- [ ] 지역 및 개별 업적 조건이 의도대로 동작한다.

## 기존 레벨 수정 시 주의

- 퍼즐 에셋 이름은 저장 키로 사용되므로 가능하면 변경하지 않는다.
- 타일 비용을 수정하면 모든 레벨의 `ans`, 임계값, 비용 업적을 다시 계산한다.
- `answerMap`만 수정해도 런타임 초기 맵은 달라지지 않는다.
- `map`만 수정하면 기존 `answerMap`과 `ans`가 낡은 정보가 될 수 있다.
- Unity 밖에서 Odin 직렬화 노드를 직접 편집하지 않는다.
- `.meta` 파일 삭제나 재생성으로 GUID가 바뀌면 지역 데이터와 튜토리얼 참조가 끊길 수 있다.
